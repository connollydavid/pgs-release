Development: Upstream Patch Series

21 patches maintained against FFmpeg master for upstream submission. Organised into 6 independent series (A–F) that can be reviewed in parallel. Also available rebased on the 8.0.1 release branch.

Approach

Prior art. Two attempts stalled. Coza (2022) put libass in libavcodec — a dependency violation. softworkz (2021–2025) proposed full subtitle filter infrastructure across 25 patches and 89 files; design questions remain unresolved.

This series takes the minimal path. libass (rendering) and Tesseract (OCR) already live in libavfilter. Small utility APIs there, called from fftools in the same pattern as sub2video, handle both directions. No filter infrastructure. No AVFrame changes.

Patches

Branch: pgs-series. Series A/B/C are independent and reviewable in parallel. D depends on all three. E and F are independent additions.

D fftools Orchestration · 4 patches

  • Rect splitting, animation detection, event coalescing
E GIF Encoder · 1 patch
  • Direct RGBA-to-GIF encoding via av_quantize_* and ff_palette_map_apply
  • Configurable quantizer (Median Cut default) and dithering (Floyd-Steinberg default)
F OCR (Bitmap to Text) · 2 patches
  • Tesseract OCR integration via avfilter_subtitle_ocr_* API
  • Bitmap deduplication for PGS fade sequences; PSM fallback for RTL scripts
  • Grayscale conversion (binary extraction for DVD, luminance-alpha for PGS)

OCR details

Tested across 114 languages with UDHR Article 1 roundtrip (text → PGS → text). 105 pass, 9 fail (Tesseract training data limitations). Bitmap deduplication skips OCR on palette-only changes (PGS fade sequences). PSM fallback handles RTL and complex scripts. Per-language preserve_interword_spaces tuning prevents word merging in Arabic, Hebrew, and Persian.

Spec Compliance

Decoder model grounded in the Panasonic and Sony HDMV patents (US20090185789A1, US8638861B2, US7620297B2). Buffer sizes, transfer rates, and palette limits taken from the patents, not reverse-engineered from player behaviour.

SUPer by cubicibo used as a hardware-validated reference.

Series Evolution

This patch series has been through four major iterations, each informed by testing, review, and upstream FFmpeg conventions. The full history is preserved as git tags.

v0 Proof of concept
  • PGS encoder only, ported from PunkGraphicStream (Java/JNI)
  • NeuQuant quantization from pngnq, hardcoded to 256 colors
  • No animation, no text-to-bitmap pipeline, no FATE tests
  • Established the patent-grounded specification (docs/pgs-specification.md)
v1 Feature-complete draft
  • 21 patches across 7 phases — encoder, quantization API, text-to-bitmap, animation, GIF, OCR
  • Public avfilter_subtitle_render_* and avfilter_subtitle_ocr_* APIs in libavfilter
  • Palette animation via composition state machine, region-weighted quantization for karaoke
  • Three quantization algorithms (NeuQuant, Median Cut, ELBG) unified under av_quantize_*
  • OCR roundtrip tested across 114 languages (105 pass, 92%)
v2 First stable release · 17 patches
  • Consolidated from 21 to 17 patches (squashed ELBG move + algo, region-weighted API + usage, quantize_method into encoder)
  • PGS decoder model compliance: DTS/PTS timing, coded data buffer validation, decoded object buffer tracking, palette optimization, object versioning
  • GIF RGBA palette mapping fix (sorted palette mismatch)
  • Released as n8.0.1-pgs.0 on FFmpeg 8.0.1 — first stable build on an official release branch
  • All upstream FATE tests pass, 8 new FATE tests
v3 Upstream review rework · 14 patches
  • End-to-end upstream suitability audit: 8 critical issues, 10 significant, all resolved
  • Renderer and OCR stay in libavfilter as internal ff_sub_render_* / ff_sub_ocr_* — no public API commitment, no version bumps
  • Orchestration moved to fftools: ffmpeg_enc_sub.c (text-to-bitmap), ffmpeg_dec_sub.c (bitmap-to-text), following the sub2video precedent
  • Eliminated #include .c pattern — animation utilities in proper ffmpeg_sub_util.h header
  • ABI-safe enum ordering (AV_QUANTIZE_NEUQUANT, MEDIAN_CUT, ELBG, NB)
  • FF-prefixed palettemap structs (FFColorInfo, FFColorNode, etc.)
  • ELBG move split from Median Cut for independent review
  • No private header imports from fftools, no unrelated cosmetic changes
